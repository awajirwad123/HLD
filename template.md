Topic/

---

### File Content Expectations

1. architecture.md
   - Explain the concept in depth
   - Include diagrams (ASCII or description)
   - Real-world architecture examples
   - Trade-offs and design decisions

2. hands_on.md
   - Practical examples (preferably Python/FastAPI)
   - Small implementation or simulation
   - API design examples if relevant

3. notes.md
   - Topic Explanations
   - revision notes
   - Key formulas / definitions
   - Important comparisons

4. quick-fire-questions.md
   - Rapid interview questions (short answers)
   - Useful for revision

5. tricky_interview_questions.md
   - Deep-dive interview questions
   - Edge cases
   - Follow-up questions asked by interviewers
